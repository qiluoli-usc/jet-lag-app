# Jet Lag App — 第一阶段复核（2026-03-16）

## 结论

第一阶段已经完成大部分基础设施建设，但按“是否已经完全收口并可签收”来判断，当前仍不能算全部完成。

核心原因不是认证或持久化未落地，而是：

1. iPhone 端推送通知还没有完成真实设备闭环验证。
2. iPhone 端后台定位在 Expo Go 下仍是前台降级方案，不是真正的后台定位能力。
3. 第一阶段文档仍把上面两项标记为已完成，和当前仓库的真实状态不一致。

## 本次复核时已确认完成的部分

以下能力已经在代码和自动化脚本层面落地，并可视为第一阶段已基本完成：

- 用户认证：注册、登录、JWT、密码校验、登录态持久化。
- 用户身份绑定：房间内玩家身份会绑定到登录用户，HTTP / WebSocket 都做了归属校验。
- SQLite 持久化：用户、房间、事件数据可重启恢复。
- 基础 API 安全：主要入口已有输入校验和可选鉴权。
- 移动端认证集成：移动端已强制登录后再进入首页和房间。
- 服务端推送事件链：关键事件可以生成推送 payload，并完成注册/投递链路测试。

本次复核重新执行并通过的脚本：

- `npm.cmd run smoke`
- `npm.cmd run test:task5`
- `npm.cmd run test:task6`
- `npm.cmd run test:task7`

## 仍未完全收口的差距

### 1. 推送通知：服务端已通，iPhone 真机闭环未完成

当前状态：

- 服务端已经支持 `/push/register`，并能在关键房间事件后生成通知 payload。
- 自动化测试 `task6` 已覆盖“注册 token + 事件触发 + payload 可读”。

仍未完成的地方：

- 移动端获取 Expo Push Token 依赖 `EAS projectId`。
- 如果没有 `projectId`，移动端会直接放弃获取 token。
- 这意味着“服务端可发通知”并不等于“iPhone 真机当前就能稳定收到通知”。
- 仓库当前 README 也已经明确说明：Expo Go 下远程推送不完整，完整验证需要 development build。

复核判断：

- 这一项目前只能算“后端与部分客户端接入完成”，还不能算“第一阶段推送通知能力完整签收”。

### 2. 后台定位：iPhone Expo Go 下仍为前台降级

当前状态：

- iOS 原生权限文案已写入 `mobile/app.json`。
- 位置上报逻辑已经支持高精度定位和跟踪上下文恢复。

仍未完成的地方：

- 在 iPhone 的 Expo Go / store client 环境下，代码会主动切换为 `foreground` 模式。
- 这意味着应用退到后台后，当前仓库默认测试路径并不能证明“后台定位已交付”。
- 仓库 README 同样已经明确写明：iPhone 背景定位在 Expo Go 下并不完整。

复核判断：

- 这一项目前只能算“已为 iOS 原生能力做好代码准备，并在 Expo Go 提供兼容降级”，还不能算“后台定位能力完全完成”。

### 3. 阶段一文档状态与真实实现口径不一致

当前状态：

- `docs/PHASE_1_TASKS_2026-03-14.md` 仍将“后台定位”“推送通知”标为已完成。
- `docs/PHASE_1_IMPLEMENTATION_PLAN_2026-03-14.md` 仍将“认证接入后后台定位和推送通知不受影响”作为验收描述。

问题：

- 当前仓库自己的 README 已经承认 Expo Go / iPhone 下这两项仍有限制。
- 因此，阶段一计划/任务文档和当前真实可交付状态之间存在偏差。

复核判断：

- 如果要正式关闭第一阶段，必须补一次文档同步，明确“哪些能力已经完成，哪些能力需要 development build 真机验证后才算完成”。

## 建议的第一阶段收口标准

要把第一阶段从“基本完成”推进到“可以签收”，建议按下面的顺序补齐：

1. 完成 iOS development build 路径，而不是继续只依赖 Expo Go。
2. 配置并验证 `EAS projectId`、推送 token 获取和 iPhone 真机通知送达。
3. 在 iPhone development build 上验证后台定位在退到后台后仍持续上报。
4. 将 `docs/PHASE_1_TASKS_2026-03-14.md` 与 `docs/PHASE_1_IMPLEMENTATION_PLAN_2026-03-14.md` 同步到真实状态。

## 当前建议结论

第一阶段的大头已经完成，尤其是认证、持久化、身份绑定、移动端认证接入和服务端事件链，这些都已经具备稳定基础。

但如果以“是否已经全部搞定”为标准，当前仍有两项真实设备能力和一项文档同步工作需要继续完成：

- iPhone 真机推送闭环
- iPhone 真后台定位闭环
- 第一阶段文档同步
